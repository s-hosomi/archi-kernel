# archi-kernel 設計方針

建築シミュレーション専用の B-rep ジオメトリカーネル。本ドキュメントは設計判断とその根拠の記録である。

- 初版 (2026-06-12): 着手前の調査・方針議論
- v2 (2026-06-12): Phase 0 完了後、本格実装前の集中調査(先行カーネル 5 系統・頑健性戦略・ブーリアンアルゴリズム・建築実務要件・Rust API 設計)と、3 つの独立設計案+敵対的レビューによる統合改訂

## 1. 背景: なぜ「汎用 B-rep カーネルのフルスクラッチ」をしないか

CAD カーネル (幾何カーネル) は B-rep (境界表現) で立体を表し、ブーリアン演算・フィレット・オフセット等を提供するエンジン。商用は Parasolid (1985〜, SolidWorks/NX/Onshape が採用)・ACIS の二強、OSS では Open CASCADE がほぼ唯一の実用品。

汎用カーネルをフルスクラッチする場合の壁:

1. **SSI (曲面×曲面交線)** — NURBS 曲面同士の交線は閉形式で解けず数値追跡 (marching) が必要。接触・接線交差・微小ループの検出漏れが即バグになる
2. **ブーリアンの頑健性** — アルゴリズム自体は古典的 (交線計算 → 面分割 → 内外分類 → 縫い直し) だが、浮動小数点下の「ぴったり接する面」「ほぼ平行な交差」処理に数十年分のノウハウが必要。Open CASCADE ですら失敗が日常的
3. **フィレット/オフセット** — ブレンド面が隣接面を食い潰すケースの処理が地獄
4. **検証の帰納性** — FEM は解析解と照合して正しさを証明できるが、ブーリアンの頑健性は「失敗例が来るたびに直す」しかない

規模感: Parasolid は数百人年。Rust 製 OSS (truck, Fornjot) は数年開発してもブーリアンが限定的。**汎用路線は個人〜小チームでは勝負にならない**。

調査による裏付け (v2):
- **web-ifc** (ThatOpen) は本カーネルと同一ドメイン (押し出しの和−開口の差) をメッシュベース fuzzy boolean で長年揉んだ結果、開口欠落・内外反転・多数開口での無限ループに今も悩まされ、トレランスを 0.1mm 級・多数・実行時可変にしてようやく安定させている。解析曲面+閉形式交差+単一絶対トレランスという本方針は、この泥沼をそもそも回避する側にある
- **Fornjot** は本カーネルと同様の曲面限定 (line/circle) から出発しながら、(a) プリミティブ対ごとの演算の組合せ爆発、(b) トポロジー/ジオメトリ分離がリファクタで崩れて半年の作り直し、を経て停滞した。限定で SSI は回避できるが、**組合せ分岐・分離規律の維持・ブーリアンの非局所性・共平面処理は回避できない** — 本設計はこの 4 点に明示的に備える

## 2. 採用方針: 建築ドメイン特化の解析曲面カーネル

### 2.1 鍵となる観察

建築躯体に現れる曲面は実質2種類:

| 曲面 | 用途 | 割合感 |
|------|------|--------|
| 平面 | 壁・床・梁・柱 (矩形/H形断面) | 9割以上 |
| 円筒面 | 円形柱、杭、配管、ボイド | 残りほぼ全部 |
| 円錐・球面 | 杭先端、特殊屋根 | ごく稀 (将来対応) |

この範囲なら**全ての曲面交差が閉形式で解ける**:

- 平面 × 平面 = 直線 (連立一次)
- 平面 × 円筒 = 円 / 楕円 / 平行線 0〜2 本 / 接線 (全ケース解析解)
- 円筒 × 円筒 = 4次空間曲線 — **非対応と割り切る**。4次交線の座標は代数的数になり有理数でも閉じないため、この割り切りは頑健性戦略としても正当。実需 (斜め杭・配管交差) が出たらメッシュ近似を最終手段とする二段構え。出現頻度は Phase 2 完了時に実 ST-Bridge データで計測する

数値 marching が不要になるため、汎用カーネル最大の不確実性 (SSI の頑健性) が消える。

### 2.2 トレランス体系 — 単純だが「これで頑健」ではない

建築は寸法レンジが固定 (1e-3 m 〜 1e2 m、SI 単位 m で計算) のため、スケール適応型トレランスは不要で、**全体で絶対トレランス 1 本** (`Tol::length = 1e-6 m`) を基本とする。

ただし正直に書く: tolerant modeling には **ε 比較の非推移性** (A≈B, B≈C でも A≉C) と**トレランス汚染**という構造的欠陥があり、建築の格子配置 (多面が一点に集まる) では確実に顔を出す。「ε 1 本だから頑健」ではなく、「ε 1 本でも破綻しないよう検証 (§7) と縮退処理 (§4) で守る」が正確な姿勢である。これを踏まえた三層方針:

1. **平面パート**: 立ち上げ期 (Phase 1〜) は tolerant。将来の厳密化 (indirect predicates) は臨界経路外の研究フェーズ (§10 Phase 8) とし、データ構造の接合面だけ先に確保する (§3.4)
2. **円筒パート**: 恒久 tolerant (円筒交点は無理数で厳密化が原理的に不可能)
3. **隔離**: 円筒を含む部材は部材単位の「tolerant ゾーン」として識別し、汚染を閉じ込める

運用原則:

- 全ての幾何述語は `Tol` を経由し、float 直接比較を禁止
- 内外判定は最初から 3 値 (`Sign3: Below / On / Above`)。建築は「ぴったり一致が常態」(壁面と床面が同一平面、開口が壁端に一致) のため、ON は第一級
- **射影を伴う判定 (2.5D 帰着の平行判定など) は角度でなく長さに換算する**: `sinθ × L_max ≤ Tol::length`。cosine の `Tol::angular` 流用は ±1 近傍で 2 次写像になり、建築スパン上限では mm 級の射影ずれを素通りさせる
- **平面の正準化 (canonical plane store)**: 別部材から独立に導出された「同一のはずの平面」の係数は ulp 単位でずれる。GeomStore への挿入時に Tol で同一視して一意の参照に統合する。ε は消えるのではなく入口に移動する — これが共平面検出・sibling 対応づけ・将来の厳密化すべての前提

### 2.3 アーキテクチャ: CSG 履歴 + 遅延 B-rep 評価

建築躯体は「押し出しの和 − 開口の差」という浅い CSG 木で完全に表現できる。IFC の開口表現 (IfcRelVoidsElement → 暗黙ブーリアン差) および国交省積算基準の柱優先内法計算と構造的に一致しており、先行例 (OpenSCAD / FreeCAD / IFC) の評価パイプラインとも整合する。

- **正本は CSG 木** (部材 = 断面押し出し + 開口リスト) で持つ
- B-rep は表示・断面・メッシュ生成時に**遅延評価** (push-dirty / pull-clean) で構築
- 評価失敗は「データ破壊」でなく「部材単位の検出可能な一時エラー」。前回の有効 B-rep を保持して表示は継続 (FreeCAD 2 パスリコンピュート流)。なお閉形式演算は決定的なので「リトライで直る」わけではない — 価値は「壊れない・原因が特定できる」ことにある

詳細は §5。

## 3. データ構造

### 3.1 モジュール構成 (単一クレート)

ワークスペース分割は外部利用者が実在するまで行わない。モジュール境界と CI 検査で依存の向きを守る。

```
src/
├── tolerance.rs      Tol, Sign3                                  [既存]
├── primitives/       解析プリミティブ (Plane/Cylinder/Line3/…)    [既存]
├── intersect/        閉形式交差 + 組合せ表テスト基盤               [既存+]
├── predicate/        頑健述語の単一窓口 (§3.5)                    [Phase 3a]
├── topo/             トポロジー (幾何を一切持たない)               [Phase 1]
├── geom/             GeomStore (平面正準化を含む幾何ストア)        [Phase 1]
├── profile/          2D 断面 (Rect / H形 / 円形)                  [Phase 2]
├── build/            押し出し構築                                 [Phase 2]
├── boolean/          半空間カット / 2.5D 差 / 共平面分岐 / 2D 多角形 [Phase 3]
├── section/          断面図出力                                   [Phase 4]
├── csg/              CsgNode / Member / 遅延評価                  [Phase 1 から器のみ]
├── mass/             質量特性                                     [Phase 5]
└── tess/             テッセレーション                              [Phase 6]
```

### 3.2 不変条件: トポロジーは幾何を持たない

**トポロジー層 (`topo/`) はジオメトリ座標を一切持たず、`geom/` の GeomStore への ID 参照のみを保持する。** Fornjot は同じ分離を当初持ちながらリファクタで崩し、半年の作り直しに至った (#2116)。この不変条件はディレクトリ物理分離+ CI の import 検査 (topo が math・primitives を import しないこと) で機械的に守る。実行時テストでは型レベルの性質は検証できない。

### 3.3 トポロジー (half-edge)

階層は `Solid → Shell → Face → Loop → HalfEdge → Vertex`。商用 3 カーネル (Parasolid=fin / ACIS=coedge / OCCT) はいずれも半エッジ系であり、建築の面共有・開口・複数ソリッド・干渉チェックは **2-多様体・向き付き閉シェルの独立ソリッド集合**で表現できる。非多様体 (radial-edge) は部屋=cell を第一級に持つ熱/エネルギー解析へ拡張する場合のみ必要で、現スコープ外と明記する。

設計要点:

- **自前の世代付き arena** (`Id{idx, gen}`、~50 行)。具体型で持ち、トレイト抽象はしない (抽象層はバグの隠れ場所)。依存ゼロを維持
- **HalfEdge は `boundary: [f64; 2]` (曲線パラメータ区間) を持つ** (Fornjot 忠実)。閉曲線 (円・楕円) 上の 2 頂点を結ぶ弧は 2 通りあり、区間なしには弧を一意に表現できない。向きは区間の大小で表す。直線のトリムも一様に扱える
- twin はフィールドに持たず、sibling (同一 curve・逆 boundary) を検証時に対応づける。対応づけは総当たり O(n²) を避け空間ハッシュで O(n log n) 化する
- 上方向の逆参照 (HalfEdge→Loop 等) は持たない。face 分割のたびに整合維持コストが生じる便利フィールドは、必要が実証されてから足す
- 面の向き (front/back) はトポロジー側に持ち、内外判定は既存の `signed_distance` + `Sign3` を再利用する

### 3.4 頂点表現 — 接合面だけ確保する

頂点の幾何は GeomStore 側に `#[non_exhaustive]` な enum で置く:

```rust
#[non_exhaustive]
pub enum VertexGeom {
    Explicit(/* 座標 */),
    // 将来: LinePlane / ThreePlanes (3平面交点の記号表現、厳密パス用)
}
```

当面は `Explicit` のみ実装する。**enum を空けただけでは厳密化の道は開かない** — 重要なのは述語窓口 (§3.5) のシグネチャが `&VertexGeom` を受ける形になっていることであり、座標 `[f64;3]` を要求する述語 API にした瞬間に記号表現は飾りになる。厳密パス自体 (indirect predicates の Rust 実装) は数ヶ月級のサブプロジェクトであり、臨界経路外 (§10 Phase 8) とする。

### 3.5 述語の単一窓口

幾何述語は `predicate/` モジュールの単一窓口に集約し、ブーリアン実装に直接埋め込まない。窓口は意味論の異なる 2 系統を明示的に分ける:

1. **Tol ベースの 3 値分類** (`Sign3` を返す) — 「設計意図として一致か」を判定する。ON 帯は仕様であり、建築の共平面常態を扱う主役
2. **厳密符号** (Phase 3a で `robust` クレート = Shewchuk orient3d を導入) — ON 帯の**外**での取り違え (真の丸め誤差による Above/Below の誤判定) を防ぐ。非推移性は ε に由来するため厳密述語では消えないが、分類の土台が揺れないことを保証する

## 4. ブーリアン・断面アルゴリズム

### 4.1 処理階層 (易しい順に基盤を積む)

```
[3a 基盤]   ソリッド × 半空間カット
            → 内外分類器・面分割器が生まれる。出力は必ず watertight (切断面に蓋)。
              断面図 (§6-3) はこの ON 部分面抽出として得られる。差より易しい
[3b 本命]   prismatic 差 = 2D 断面ブーリアン + 軸方向区間演算
            → 建築の圧倒的多数。IfcOpenShell が 2〜3 倍速・アーティファクト減を実証済み
[3c 拡張]   2D 円弧対応 (矩形×円形ボイド) → 汎用 3D 4段階フォールバック
            → 3D フルパス実装までは EvalError::Unsupported3dBoolean (データを壊さない明示エラー)
```

### 4.2 2.5D 帰着は「共通 prismatic 方向」で判定する

判定は「両者の押し出し軸が平行か」では**ない**。「**共通の prismatic 方向が存在するか**」である。直方体は 3 軸すべてに prismatic なので、矩形柱 × 梁の直交差 (最初のマイルストーン、および数量拾いの柱優先差) は梁軸を共通方向に取って 2.5D に落ちる。円形柱 × 梁は共通方向がなく General 落ち (当面未対応と明記)。

結果の構築は「重なり区間で帯に切って再押し出し・縫合」では**なく**、「軸方向の区間 × 2D アレンジメント (分割領域集合) のグリッドから一括構築」とする。帯方式は帯境界に部分重なりの共平面蓋ペアを生み、避けたはずの古典 Imprint+共平面分類が縫合段に再来する。

2D 多角形エンジン (`boolean/polygon2d`) は事実上の第二のカーネルである (Clipper2 が長年磨いた領域)。線分のみ → 円弧 (矩形×円形ボイド限定) の順に段階実装し、円弧×円弧の接線縮退は当面スコープ外とする。

### 4.3 共平面ケース — SoS 不採用、反例つき真理値表で固定

汎用の記号摂動 (SoS) は平坦領域に sliver を生むため不採用。建築の縮退は種類が限定列挙可能 (共平面 face / 共線エッジ / 一致頂点) なので、**演算 (差・積・和) × 自分/相手 × 法線同/逆の全組合せを、各セルに反例つきの真理値表として実装前に確定**し、`boolean/coplanar.rs` に固定する。

規則を取り違えやすいので正誤判定例を仕様に含める (差 A−B で A の面が ∂B 上に乗る場合):

- **法線同方向 → 捨てる**。反例: A=[0,1]³, B=[0,1]²×[0,0.5] (貫通差)。両者の底面 z=0 は法線同方向で一致し、正しい結果に z=0 の面はない
- **法線逆方向 → 残す**。反例: B=[0,1]²×[−0.5,0] (下に純接触)。A−B = A であり、A の底面は残らなければならない

この 2 反例を `tests/coplanar.rs` の最初の 2 ケースにする。

### 4.4 差だけでなく積・和も最初から仕様に含める

内外分類器は差・積・和で共有でき、残否表が違うだけである。**積 (∩) は体積恒等式 `V(A−B) = V(A) − V(A∩B)` の検算に、和 (∪) は多数開口の一括融合に必須**であり、「差が圧倒的に多い」ことは積・和を仕様から落とす理由にならない。

### 4.5 fail-safe (web-ifc の教訓を最初から内包)

- 多数開口は和オペランドとして一括融合してから 1 回で差し引く (web-ifc BOOLEAN_UNION_THRESHOLD の内包)
- 評価に複雑度上限を設け、超過は `EvalError::ComplexityLimit` として部材単位に隔離
- 失敗した開口を持つ部材だけが赤表示になり、他は正常 — 局所失敗隔離

## 5. CSG 木と遅延評価

### 5.1 ノード語彙

```rust
#[non_exhaustive]
pub enum CsgNode {
    Extrude { profile, axis, length },
    Union(Vec<CsgNode>),
    /// IFC IfcRelVoidsElement 相当の意味的開口。型枠面積・開口面積の算定が
    /// 木走査で完結するために汎用 Difference とは別に必要
    OpeningSubtraction { base, openings: Vec<(OpeningId, Opening)> },
    /// 数量拾いの優先順位控除 (柱優先→大梁→小梁→壁/床)。控除は「開口」では
    /// ないため OpeningSubtraction で代用しない (型枠面積の木走査が誤る)
    Clip { base, clippers: Vec<StableId>, rule: ClipRule },
    /// 汎用ブーリアン (仕口・斜め切り欠き)
    Difference { positive, negative },
}
```

- **stable id**: 開口・部材は安定 id (`u64` newtype) で識別し、B-rep の face インデックスを外部参照に使わない (トポロジカル命名問題の構造的回避)。採番は利用側 (アダプタ/アプリ層) の責務で、カーネルは opaque に扱う
- **部材間依存はモデル層の DAG で管理する**。柱優先控除は部材間ブーリアンであり、柱が動いたら梁の dirty が立つ仕組みは Member 単体のフラグでは作れない (FreeCAD 流のトポロジカルソート再評価)

### 5.2 評価とキャッシュ

- push-dirty / pull-clean、**部材単位**キャッシュ (face 単位は建築では再利用率が低い)
- **キャッシュキーには Tol を含める** (異なる Tol での再評価が古い結果を返さないように)
- **GeomStore は Member が所有し、評価のたびに作り直す** (B-rep は使い捨て)。部材間ブーリアンは入力 read-only・出力新規ストア。幾何共有 (正準化平面の共有) は部材内に限定される
- 失敗時は `prev_valid` (前回の有効 B-rep) を保持して表示継続。`EvalError` は機械可読に細分化し (どの開口・どの縮退種別か)、Defect は座標・OpeningId など自己完結した記述を持つ (破棄済み arena の Id を持たない)

## 6. 必要な演算 (建築シミュレーションからの逆算)

1. **押し出しソリッド生成** — 2D 断面 (H形・矩形・円形) を軸線に沿って押し出す。ST-Bridge の部材表現 (軸線 2 ノード + 断面参照) に直接対応
2. **ブーリアン差** — 壁・床の開口、部材端部の切り欠き、数量拾いの優先順位控除 (国交省「公共建築数量積算基準」の柱優先ルール)。和・積は §4.4 のとおり仕様に含める
3. **平面断面カット** — ソリッド × 平面 = 伏図・軸組図・断面図。切断平面が部材 face とぴったり一致するのが常態 (通り芯・スラブ天端) なので、共平面時に面を含めるか否かの規約を出力仕様として固定する。出力は外周+穴 (ドーナツ) を表現できる `Polygon2d` (穴付き)
4. **質量特性** — 体積・重心・断面性能。地震用質量 (令88条) と数量拾い (コンクリート体積・型枠面積) をジオメトリから直接算定
5. **テッセレーション** — 表示用 (Three.js) + FEM メッシュ生成用。watertight 保証
6. **干渉チェック** — ハード干渉 (体積重複) とスリーブ検証 (径・位置ルール、Cylinder プリミティブを直接活用)

機械 CAD の難所 (フィレット・シェル化・ロフト) は**丸ごと不要**。

## 7. 検証戦略

「失敗を必ず検出できる」ことを設計の合否条件とする。検証は後処理でなく評価パイプラインの一部 (`validate()` を一級 API に)。

- **オイラー標数 (リング項込み)**: `V − E + F − (L − F) = 2(S − G)`(L = 全ループ数)。開口付きの面は interior loop を持つため、リング項なしの式は壁+窓という最初の実用ケースで即破綻する。G の期待値照合は貫通が自明なケース (軸平行・縁非接触の開口) に限定し、一般ケースは独立に数えた量の整合検査に留める — G の一般予測はブーリアンを解くのと同程度に難しい
- **watertight**: 各エッジがちょうど 2 つの半エッジ (sibling ペア完全性)。テッセレーション結果は各エッジちょうど 2 三角形
- **体積恒等式**: `V(A−B) = V(A) − V(A∩B)` をランダム配置で大量検証 (proptest)。許容誤差は Tol::length と関与面積から導出して明示する
- **解析解照合**: 交線・交差体積を閉形式の手計算と照合 (`tests/analytic.rs`)。既存文化を継続
- **共平面スイート**: §4.3 の反例 2 件を起点に、ぴったり接触・端一致を専用テスト群として整備。**web-ifc の既知の失敗 issue (内外反転 #63、開口欠落 #346/#398、多数開口無限ループ #535/#1786) を回帰テストの種にする**
- **縮退スイート**: 近平行・近垂直 (`Tol::angular` の cosine 流用の境界挙動)
- **組合せ表テスト**: 交差・分類の全分岐を enum で網羅固定し、プリミティブ追加時の組合せ爆発を毎回検知 (Fornjot #2118 対策)
- **数量拾い E2E**: 矩形柱 2 本 + RC 大梁 1 本の柱優先差で体積・型枠面積を検証値照合 — 商用デモの受入テスト
- 検証コストは 2 段階: 編集中は軽量 (watertight + オイラー)、確定時 (保存・数量拾い・FEM 出力) に完全 (体積恒等式・自己交差)

数値テストのリテラルには `f64` 型注釈を付ける (fem の規約を踏襲)。

## 8. 公開 API・依存方針

- **コンストラクタはパニックしない**: `assert!` を全廃し `Result` を返す (Phase 0.5 で既存 API を v0.2.0 として改修)
- **サードパーティの線形代数型を公開 API に露出しない**: 外部クレートのマイナー更新が本クレートのメジャー更新を強制するため。v0.2.0 で nalgebra を**自前の最小 math モジュール** (`Point3`/`Vec3`/`Unit3`、必要演算は dot/cross/norm 程度) で置換し完全に除去した — 変換層のノイズがなく、依存ゼロが文字通り成立する
- **依存方針 (明文化)**: 実行時依存ゼロ + Phase 3a から `robust` (Shewchuk 厳密述語、述語窓口内に限定)。serde は optional feature (未使用なら依存に現れない)。エラー型は手書き impl とし thiserror は採らない。「依存ゼロの純粋ライブラリ」(§11) が文字通り成立
- **serde は自前型に直接 derive**する (自前型はワイヤフォーマットを自己決定でき、外部クレートの内部表現に結合しない)
- 全公開 enum に `#[non_exhaustive]` (円錐対応等の variant 追加を semver-safe に)。CI に cargo-semver-checks
- **wasm**: コアは wasm 互換を維持 (スレッド・FS 非依存)。wasm-bindgen ラッパは将来の別クレートとし、境界はフラットな f64 配列で渡す
- 単位は SI (m, rad) のみ。**ST-Bridge は mm 単位**なので、×10⁻³ の変換はアダプタ層 (fem-io 側) に閉じ込める

## 11. fem リポジトリとの関係

- 本カーネルは依存最小の純粋ライブラリ (Clean Architecture の Entities 層相当)。fem → archi-kernel の一方向依存のみ
- fem 側からは `Cargo.toml` の git 依存で取り込む。ローカル並行開発時は `[patch]` / path 依存
- 想定パイプライン: ST-Bridge (fem-io、**mm→m 変換はここ**) → ソリッドモデル → 干渉チェック・数量拾い (コンクリート体積・型枠面積)・計算書躯体図。数量拾いは意匠事務所向け商用化で直接価値になる

## 12. コーディング規約

- 標準 rustfmt、clippy 警告ゼロ (`-D warnings`)、公開型に rustdoc
- 単位は SI (m, rad) のみ。表示単位変換は利用側 (adapter) の責務
- float 直接比較禁止 — 幾何述語は必ず `Tol` 経由、述語は `predicate/` 窓口に集約
- コンストラクタはパニックしない (Result を返す)。公開 API はすべて自前型 (math モジュール)
- トポロジー層は幾何を持たない (CI で機械検査)
- テスト数値リテラルに `f64` 型注釈、許容誤差を明示
- 公開 enum に `#[non_exhaustive]`

## 13. 未解決の論点 (楽観で塞がず明示的に残す)

1. 円筒×円筒 (斜め杭・配管交差) の実データ出現頻度 — Phase 2 完了時に計測。実需が出たらメッシュ近似フォールバックの設計が必要 (marching 排除原則との緊張)
2. 2.5D に落ちない差 (斜め仕口) の実頻度 — 同上。3D フルパスの投資判断材料
3. 円弧×円弧の 2D ブーリアン縮退 (接線・端点一致) — Phase 3c は矩形×円形ボイドに限定
4. 純接触 (共有体積ゼロ) の差の意味論 — 薄面を残すか消すかは用途で分岐 (取り合い検出 vs 数量拾い)。フラグ切替とし既定は「何も削らない」
5. 1 エッジに 3 面以上が一致接触する局所非多様体 — 共平面スイートで実例を収集して判断
6. 検証コストとインタラクティブ性の境界 — 2 段階検証で運用するが「確定」の境界は GUI 設計待ち
7. 平面厳密パス (Phase 8) の採算 — indirect predicates の Rust 移植は数ヶ月級。Phase 5 の体積恒等式 proptest で tolerant パスの実際の失敗率を測ってから判断

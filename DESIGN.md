# archi-kernel 設計方針

建築シミュレーション専用の B-rep ジオメトリカーネル。本ドキュメントは着手前の調査・方針議論 (2026-06-12) の記録であり、設計判断の根拠を残すことを目的とする。

## 1. 背景: なぜ「汎用 B-rep カーネルのフルスクラッチ」をしないか

CAD カーネル (幾何カーネル) は B-rep (境界表現) で立体を表し、ブーリアン演算・フィレット・オフセット等を提供するエンジン。商用は Parasolid (1985〜, SolidWorks/NX/Onshape が採用)・ACIS の二強、OSS では Open CASCADE がほぼ唯一の実用品。

汎用カーネルをフルスクラッチする場合の壁:

1. **SSI (曲面×曲面交線)** — NURBS 曲面同士の交線は閉形式で解けず数値追跡 (marching) が必要。接触・接線交差・微小ループの検出漏れが即バグになる
2. **ブーリアンの頑健性** — アルゴリズム自体は古典的 (交線計算 → 面分割 → 内外分類 → 縫い直し) だが、浮動小数点下の「ぴったり接する面」「ほぼ平行な交差」処理に数十年分のノウハウが必要。Open CASCADE ですら失敗が日常的
3. **フィレット/オフセット** — ブレンド面が隣接面を食い潰すケースの処理が地獄
4. **検証の帰納性** — FEM は解析解と照合して正しさを証明できるが、ブーリアンの頑健性は「失敗例が来るたびに直す」しかない

規模感: Parasolid は数百人年。Rust 製 OSS (truck, Fornjot) は数年開発してもブーリアンが限定的。**汎用路線は個人〜小チームでは勝負にならない**。

## 2. 採用方針: 建築ドメイン特化の解析曲面カーネル

### 2.1 鍵となる観察

建築躯体に現れる曲面は実質2種類:

| 曲面 | 用途 | 割合感 |
|------|------|--------|
| 平面 | 壁・床・梁・柱 (矩形/H形断面) | 9割以上 |
| 円筒面 | 円形柱、杭、配管、ボイド | 残りほぼ全部 |
| 円錐・球面 | 杭先端、特殊屋根 | ごく稀 (将来対応) |

この範囲なら**全ての曲面交差が閉形式で解ける**:

- 平面 × 平面 = 直線 (連立一次)
- 平面 × 円筒 = 円 / 楕円 / 平行線 0〜2 本 / 接線 (全ケース解析解)
- 円筒 × 円筒 = 4次空間曲線 — **唯一の難所だが建築では稀。v0 では非対応と割り切る**

数値 marching が不要になるため、汎用カーネル最大の不確実性 (SSI の頑健性) が消える。

### 2.2 トレランス体系の単純化

建築は寸法レンジが固定 (1e-3 m 〜 1e2 m、SI 単位 m で計算) のため、**全体で絶対トレランス 1 本** (`Tol::length = 1e-6 m`) で済む。Parasolid が解いている「0.001mm の部品と 10m の金型の同居」問題は存在しない。

ただし建築は格子状配置ゆえ「**ぴったり一致が常態**」(壁面と床面が同一平面、開口が壁端に一致)。よって:

- 全ての幾何述語は `Tol` を経由し、float 直接比較を禁止
- 内外判定は最初から 3 値 (`Sign3: Below / On / Above`)。後付けにすると書き直しになる

### 2.3 アーキテクチャ: CSG 履歴 + 遅延 B-rep 評価

建築躯体は「押し出しの和 − 開口の差」という浅い CSG 木で完全に表現できる。

- **正本は CSG 木** (部材 = 断面押し出し + 開口リスト) で持つ
- B-rep は表示・断面・メッシュ生成時に**遅延評価**で構築

利点: ブーリアン失敗が「データ破壊」でなく「再評価でリトライ可能な一時エラー」になる。開口位置変更などのパラメトリック編集と自然に整合する。

## 3. 必要な演算 (建築シミュレーションからの逆算)

1. **押し出しソリッド生成** — 2D 断面 (H形・矩形・円形) を軸線に沿って押し出す。建築部材は本質的に全部これ
2. **ブーリアン差** — 壁・床の開口、部材端部の切り欠き、仕口の干渉除去。和・積より差が圧倒的に多い
3. **平面断面カット** — ソリッド × 平面 = 平面図・断面図の自動生成
4. **質量特性** — 体積・重心・断面性能。地震用質量 (令88条) をジオメトリから直接算定可能に
5. **テッセレーション** — 表示用 (Three.js) + FEM メッシュ (シェル/ソリッド要素) 生成用
6. **干渉チェック** — 部材間クラッシュ検出

機械 CAD の難所 (フィレット・シェル化・ロフト) は**丸ごと不要**。

## 4. 検証戦略

fem リポジトリと同じ「解析解照合」文化を適用する:

- **解析解照合**: 交線・交差体積を閉形式の手計算と照合 (`tests/analytic.rs`)。許容誤差を明示
- **体積恒等式**: `V(A∪B) = V(A) + V(B) − V(A∩B)` をランダム配置で大量検証 (proptest 導入予定)
- **オイラー標数**: `V − E + F = 2(S − G)` を全トポロジー操作後に検査
- **watertight 判定**: テッセレーション結果の各エッジがちょうど2三角形に共有されること
- **共平面スイート**: ぴったり接触・端一致のケースを専用テスト群として最初から整備

数値テストのリテラルには `f64` 型注釈を付ける (fem の規約を踏襲)。

## 5. 参考実装 (要調査)

| プロジェクト | 言語 | 参考になる点 |
|---|---|---|
| **web-ifc** (ThatOpen) | C++/WASM | まさに「建築特化の自前カーネル」の商用実例。IFC 形状評価のためブーリアンまで自作。設計判断の最重要参考 |
| **SolveSpace** | C++ | 個人作の小カーネル (NURBS ブーリアン搭載)。コードベースが小さく通読可能 |
| **Manifold** | C++ | メッシュベース頑健ブーリアン。「B-rep を捨ててメッシュで行く」対案の検討材料 |
| **truck** | Rust | Rust での B-rep データ構造設計の参考 |
| **Open CASCADE** | C++ | 機能の網羅リストとして。コードは参考にしない (規模が違いすぎる) |

文献: Piegl & Tiller "The NURBS Book" (本カーネルでは NURBS 不使用だが教養として)、Hoffmann "Geometric and Solid Modeling" (無料公開、B-rep ブーリアンの古典)。

## 7. fem リポジトリとの関係

- 本カーネルは依存ゼロの純粋ライブラリ (Clean Architecture の Entities 層相当)。fem → archi-kernel の一方向依存のみ
- fem 側からは `Cargo.toml` の git 依存で取り込む。ローカル並行開発時は `[patch]` / path 依存
- 想定パイプライン: ST-Bridge (fem-io) → ソリッドモデル → 干渉チェック・数量拾い (コンクリート体積・型枠面積)・計算書躯体図。数量拾いは意匠事務所向け商用化で直接価値になる

## 8. コーディング規約

- 標準 rustfmt、clippy 警告ゼロ (`-D warnings`)、公開型に rustdoc
- 単位は SI (m, rad) のみ。表示単位変換は利用側 (adapter) の責務
- float 直接比較禁止 — 幾何述語は必ず `Tol` 経由
- テスト数値リテラルに `f64` 型注釈、許容誤差を明示
